# 深模块

出自《软件设计哲学》（A Philosophy of Software Design）：

**深模块** = 小接口 + 大量实现

```
┌─────────────────────┐
│       小接口         │  ← 方法少、参数简单
├─────────────────────┤
│                     │
│                     │
│       深实现         │  ← 复杂逻辑藏在里面
│                     │
│                     │
└─────────────────────┘
```

**浅模块** = 大接口 + 少量实现（避免）

```
┌─────────────────────────────────┐
│             大接口               │  ← 方法多、参数复杂
├─────────────────────────────────┤
│            薄实现                │  ← 只是透传
└─────────────────────────────────┘
```

设计接口时，问自己：

- 方法数能不能再少？
- 参数能不能再简化？
- 还能再往里多藏点复杂度吗？
