# Stencil & Tablet — 使用规则指引（编排手册）

> **来源**：本文件是 `components.md`（物料卡）的姊妹文件。物料卡告诉你「有哪些积木」，本文件告诉你「**手里这段内容该挑哪块积木**」。
> **优先级**：本文件 > `components.md` > `template.html`。三者冲突时以本文件为准。
> **使用方式**：拿到一篇 markdown / spec / 调研笔记，按 §1 → §2 → §3 → §4 → §5 → §7 顺序决策；§6 兜底；§7 是交付前的自检闸门。

---

## 1. 核心约束（先读这五条）

1. **只能用 `components.md` 已登记的组件**。本文件不引入任何新组件、新 class、新 CSS。
2. **整篇文档骨架按 §3 的固定流水线走**，cover / toc / footer 缺一不可。
3. **映射决策的唯一依据是「输入文本的形态特征」**（条数、结构、是否有数字、是否有出处……），不要按主题/感觉挑组件。
4. **节奏预算（§4）是硬上限**，不是建议值。超限就降级到更轻的组件或合并。
5. **不确定时永远走兜底（§6）**，禁止自创。宁可朴素，不要乱套。

### 2.A 升级 / 降级信号

判断「这段内容到底是普通段落还是该升级成 tablet/stat」的具体触发条件：

- **普通要点 → 升级成 §7b tablet-row**：原文同时满足「短标题（≤8 字）+ 解释（1–2 句）+ 总条数为 2/4/6」。任一不满足就降回 §5 `<ul>`。
- **tablet → 降级回 §5 列表**：条数奇数、解释长度差距悬殊（一条 1 句一条 5 句）、缺少明确短标题。
- **数字段落 → 升级成 §8 stat**：原文至少 2 条形如「数字 + 单位 + 一句含义」的并列结构。只有 1 条就走 §5 段落或 inline `<strong>`。
- **段落 → 升级成 §6 action-bar**：原文本身就是 TL;DR / Heads up / Note 开场，或后跟 3–5 条 bullet。普通描述段不要升级。
- **普通引用 → 升级成 §12 quote-panel**：原文用 `>` 引用且带出处，且引文是整篇/整章的核心论点浓缩。日常 `>` 引用走 §5 的 `<blockquote>` 或干脆段落。
- **配图 → 升级成 §14 figure-tablet**：图本身承担「这篇文档我只记得这张」的份量（架构图、主流程图）。截图、辅助示意全部走 §13。
- **mermaid 块 → 降级**：单图超过 10 个节点 / 12 条边、内容像目录 / 接口 / 文件 / 依赖清单、或标签含复杂代码且无法确认语法安全时，改用 §10 matrix、§7b tablet-row 或 §5 列表。只有当前工具已实际生成静态 SVG / PNG 时，才能用 §14 替代 §14b；不要凭空声明已生成静态图。

## 3. 整篇文档骨架编排顺序

固定流水线，顺序不可调整：

```
<div class="page">
  <nav class="toc">                      ← §1，每篇 1 次
  <main class="main">
    <header class="cover">               ← §2，每篇 1 次
    [<div class="section-divider">]?     ← §3，可选，长文档才用
    <section id="...">                   ← §4，1..N
      <h2>...</h2>                       ← §5
      [<p class="lead">...]              ← §5 lead，可选，每节最多 1 段
      ...任意组合 §6/§7b/§8/§9/§10/§11/§12/§13/§14/§14b
    </section>
    [更多 section]
    <footer class="doc-footer">          ← §15，每篇 1 次
  </main>
</div>
```

**硬性要求**：
- 必有：`§1 toc` + `§2 cover` + 至少 1 个 `§4 section` + `§15 footer`。
- 每个 `<section>` 必须有 `id`，且 `id` 在 `§1 toc` 的某个 `<a href="#...">` 中被引用。
- `<section>` 之间不要塞裸 `<p>`、`<div>`；要写散段必须先开新 section。

## 4. 视觉节奏预算（硬上限）

| 组件 | 单篇上限 | 备注 |
|---|---|---|
| §6 action-bar | 2 | 超 2 就只留 1 个，其余降到 §5 列表 |
| §7a tablet 单块 | 0 | 永远成组用 §7b，单块视觉过薄 |
| §7b tablet-row 每组块数 | 2 / 4 / 6（偶数） | 奇数就补一块 paper 或拆掉 |
| §8 stat-row 每组块数 | 3 最佳，2 或 4 可接受 | 同一篇内 stat-row 不超过 2 组 |
| §10 matrix | 2 | 列数 3–5；超出拆成两张 |
| §12 quote-panel | 2 | 滥用会失重 |
| §13 figure | 不限 | 但相邻不得连续 ≥3 张，中间至少插一段文字或 tablet |
| §14 figure-tablet | 2 | 画框版有重量 |
| §14b mermaid-figure | 2 | 同上 |
| §14 + §14b 合计 | 3 | 画框图总量封顶 |

**节奏建议**：每屏（约 800 字）至少 1 个非段落组件（tablet / stat / figure / quote / matrix），避免变成纯文字砖墙；连续 2 个非段落组件之间至少插 1 段 `<p>`，避免视觉打架。

## 5. 配色语义公约

### 5.1 tablet 颜色（7 种）—— 按语义选，不按好看选

| 颜色 | 语义触发条件 |
|---|---|
| `tablet--orange` | 主要 / takeaway / 首选方案 / 一组里最关键的那一块 |
| `tablet--teal` | 成功 / 正面 / 推荐做法 / 正确示例 |
| `tablet--magenta` | 警示 / 反例 / 强声明 / 不要做的事 |
| `tablet--mustard` | 提醒 / 注意事项 / 需要权衡的点 |
| `tablet--sienna` | 补色 1，第 3–6 块打散节奏用 |
| `tablet--blue` | 补色 2，技术性 / 流程性条目 |
| `tablet--olive` | 补色 3，中性陈列里加一点温度 |
| `tablet--paper` | 中性 / 最低权重 / 兜底；语义不明的条目用这个 |

### 5.2 成组配色推荐（按总块数）

| 总块数 | 推荐配色 |
|---|---|
| 2 块 | `orange + teal`（主 + 正） / `orange + magenta`（主 + 反） |
| 4 块 | `orange + teal + mustard + magenta`（要点 / 正 / 注意 / 反，覆盖四象限） |
| 6 块 | `orange + teal + magenta + mustard + sienna + blue`（四象限 + 2 块补色） |
| 全 paper | 仅用于术语表 / 字段表这类纯陈列；视觉偏闷，建议至少把首块换成 orange |

### 5.3 pill 颜色（4 种 · 固定）

| 颜色 | 语义 | 错用示例 |
|---|---|---|
| `pill--yes` | 肯定 / 完成 / 支持 | ❌ 用于「主要」「重点」（那是 tablet--orange） |
| `pill--partial` | 部分 / 受限 / 警告 | ❌ 用于「注意」「提醒」（那是 tablet--mustard） |
| `pill--no` | 否定 / 阻塞 / 不支持 | ❌ 用于「反例」（那是 tablet--magenta） |
| `pill--note` | 注释 / 中性说明 / 备注 | ❌ 用于「主要 takeaway」 |

**pill 是状态符号，不是装饰**。不要为了配色好看换颜色。

## 6. 兜底策略（不匹配时怎么办）

按优先级依次尝试：

1. 内容是「单条事实 / 状态标记」→ §11 pill
2. 内容是「长附录 / 原始日志 / 可折叠」→ 原生 `<details><summary>{{标题}}</summary>{{内容}}</details>`
3. 内容是「弱语义的陈列条目」→ `tablet--paper` 走 §7b
4. 实在不匹配 → §5 基础排版（h2 / h3 / p / ul / ol）
5. 上面都不行 → **停下**，告知用户「此内容需要新组件，请扩展 `template.html` + `components.md` 后再渲染」，**禁止自创**

**禁止清单**（违反即重做）：
- 自创组件 / 自创 class / 写 inline `style="..."`
- 引入额外的 `<link rel="stylesheet">` 或 `<style>` 块
- 引入额外 JS（mermaid 已在模板内 lazy load）
- 嵌套未在 `components.md` 出现过的组合（tablet 套 tablet、figure 套 figure、quote 套 tablet……）
- 删改 `template.html` 的 `<head>`（字体、mermaid script、元数据）

## 7. 渲染前自检清单（交付闸门）

输出 HTML 前 agent 自己跑一遍。**任一不通过禁止交付。**

### 7.1 骨架与结构
- [ ] `<html lang="...">` 正确（整篇中文 `lang="zh"`；中英混排 `lang="en"` 且中文段落加 `lang="zh"`）
- [ ] `<head>` 完全沿用 `template.html`，未删除字体 / mermaid script / meta
- [ ] `<div class="page">` → `<nav class="toc">` + `<main class="main">` 完整
- [ ] cover、toc、≥1 个 section、doc-footer 都存在

### 7.2 TOC 与锚点
- [ ] TOC 每条 `<a href="#xxx">` 在正文有对应 `id="xxx"`
- [ ] 没有死链
- [ ] 章节顺序与 TOC 顺序一致

### 7.3 占位符与内容
- [ ] 所有 `{{XXX}}` 占位符已替换，无残留
- [ ] 用户内容已 HTML 转义（`&` → `&amp;`、`<` → `&lt;`、`>` → `&gt;`、`"` → `&quot;`）
- [ ] **例外**：§14b 的 mermaid 源码原样保留，未做转义
- [ ] `<a href>` 的 URL 合法、未截断

### 7.4 预算与节奏（§4）
- [ ] §6 action-bar ≤ 2
- [ ] §12 quote-panel ≤ 2
- [ ] §10 matrix ≤ 2
- [ ] §14 figure-tablet ≤ 2
- [ ] §14b mermaid-figure ≤ 2
- [ ] §14 + §14b 合计 ≤ 3
- [ ] 没有任何 tablet 单独出现（全部用 §7b 成组）
- [ ] tablet-row 每组块数为偶数
- [ ] 连续 ≥3 张 figure 之间已插入文字或 tablet 打断

### 7.5 样式与组件
- [ ] 没有任何 `style="..."` inline 样式
- [ ] 没有新增 `<link rel="stylesheet">` 或 `<style>` 块
- [ ] 没有未在 `components.md` 出现的 class
- [ ] mermaid 块用 `<pre class="mermaid">`，**未**嵌 `<code>` / 未写成 `<code class="language-mermaid">`
- [ ] mermaid 单图节点 ≤10、边 ≤12；没有把目录、接口、文件或依赖清单压成一张横向大图
- [ ] 新生成 mermaid 的节点 ID 是短 ASCII；含 `/`、`:`、`()、{}`、`[]`、逗号、空格、代码片段或中文的展示文本已放进双引号标签

### 7.6 CJK
- [ ] 中文段落 / 标题加了 `lang="zh"`
- [ ] 中文标题末尾**未**带 `。`
- [ ] 用全角标点 `，。：；！？`，未混用半角
- [ ] 中英 / 中数之间留半角空格（盘古之白）

### 7.7 无障碍
- [ ] 每个 `<img>` 都有 `alt`
- [ ] `aria-hidden="true"` 仅用于装饰性元素（`__mark`、`__sep`）
- [ ] `<table class="matrix">` 表头用了 `scope="col"` / `scope="row"`
- [ ] 链接文字不是「点这里」「more」这类无语义文字
